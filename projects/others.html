A Few Other Projects

These are few of the other projects I have made over the years. The domain of these varies from pure software based to hardcore electronics based.

All the electronics based projects used either an Arduino, or a Raspberry Pi

Spotify Playlist Generator

The purpose of this software is to automate the discovery of new music

The init code prepared a list of artists I had not heard of before, based on the artists I like.Using the Spotify API, a playlist for each of those artists was created every day (one artist per day)

A cron job was setup, to run the code once everyday,thus automatically creating a playlist for me every day

Miniature Security System

This was built while I was figuring out various tools like Mosquitto,SMTP etc.

An IR sensor connected to a microcontroller was placed near the door of a room.When someone entered the room, a 10 second video was recorded by a Raspberry Pi, and sent over email to me

The communication between the Arduino and the Raspberry Pi was through the Mosquitto service, and a SMTP was used to send the mail

Chatbot And Social Media Auto Responder

This was made using ML to automate the process of responding to trivial messages on social media platforms

A chatbot was trained using ML, based on my conversations with people on WhatsApp (WhatsApp allows you to export a chat data, which is a text file containing all your conversations for a particular group). I recovered about 50,000 conversations, and used those to train the model

A blacklist was created, which was basically a list of people to NOT auto respond to (such as professors,official school groups etc.) Using webscraping,and the Selenium driver, the interface to whatsapp was made

Hand Movement Controlled Bot

A wireless bot, whose movements were based on gestures from my hand

The communication between the controller and the reciever was over WiFi, using the ESP8266 microcontrollers

An accelerometer and gyroscope were connected to the transmitter side microcontroller and the whole system was mounted on a glove. A few other controls were also added to the glove to make the bot perform tasks other than just linear movement

The amount of tilt of the hand controlled the speed,and the direction of the tilt controlled the direction of the bot

Object based self aligning bot

Using image processing, this bot located a particular object and aligned itself with respect to the said object

This was built as a part of a larger system which required the bot to be aligned with respect to a particular object. The constraint set on the object was that it will always be of a single uniform colour (the colour itself may vary for different test cases)

Using image processing,the region of interest was narrowed down to the part of the frame which had a uniform colour for an area greater than a set threshold. Treating this area as the object, mathematical operations were run based on the extent of ROI and the centroid was found. The bot was then made to move such that the centroid of the object would shift to the centre of the frame

A camera mounted on a Raspberry Pi was used for the image processing,and a serial communication was established with the arduino controlling the motors of the bot

LAN Music Player

Made as a software to play music for social gatherings, this player allowed the user to upload music from their devices to the server, and anyone on the same network could play the song on their own device

Built strictly to play over a LAN (to not raise copyright issues on songs), the server was hosted on a Raspberry Pi. The users could go on to the webpage and play any song from the list of available ones or upload their own.

Uploading songs allowed rest of the users to access them as well, making it ideal for an easy playlist manager at parties and such

It was not a real time streaming service,hence people could listen to whatever song they wanted without disturbing other people
